# Yosegment ROS2 OctoMap 发布器 本文档介绍 Yosegment 项目的 ROS2 OctoMap 发布功能。 --- ## 概述 Yosegment 提供两种 ROS2 发布器： 1. **静态数据发布器** (`occ_zband_publisher.py`) - 从 NPZ 文件加载地图数据并发布 2. **实时发布器** (`realtime_octomap_node.py`) - 从摄像头/视频/RTSP 流/ROS2 话题实时处理并发布 --- ## 静态数据发布器 ### 功能特点 - 从 NPZ 文件加载 2.5D 地图数据 - 发布为 ROS2 OctoMap 消息 - 可选发布 OccupancyGrid 和 MarkerArray ### 使用方法 ```bash # 基本用法 python -m app.ros2.occ_zband_publisher --npz data/my_map.npz # 指定分辨率和帧 ID python -m app.ros2.occ_zband_publisher \ --npz data/my_map.npz \ --frame-id map \ --resolution 1.0 \ --rate 2.0 # 发布 OctoMap（默认） python -m app.ros2.occ_zband_publisher --npz data/my_map.npz --publish-octomap # 发布 OccupancyGrid（可选） python -m app.ros2.occ_zband_publisher --npz data/my_map.npz --publish-occupancy-grid # 发布 MarkerArray（可选） python -m app.ros2.occ_zband_publisher --npz data/my_map.npz --publish-markers ``` ### 参数 | 参数 | 说明 | 默认值 | |------|------|--------| | `--npz` | NPZ 文件路径（必需） | - | | `--frame-id` | TF 坐标系 ID | `map` | | `--resolution` | 地图分辨率（米/格） | `1.0` | | `--rate` | 发布频率（Hz） | `2.0` | | `--publish-octomap` | 发布 OctoMap | `True` | | `--publish-occupancy-grid` | 发布 OccupancyGrid | `False` | | `--publish-markers` | 发布 MarkerArray | `False` | --- ## 实时发布器 ### 功能特点 - **多输入源支持**：摄像头、视频文件、RTSP/RTMP 流、ROS2 图像话题 - **多后端支持**：ONNX（GPU/CPU）、RKNN（RK3588 NPU） - **MJPEG 流输出**：支持远程 Web 预览 - **项目目录管理**：自动保存处理结果 ### 使用方法 #### 从摄像头实时发布 ```bash # 使用默认后端（自动检测） python -m app.ros2.realtime_octomap_node --source 0 # 使用 RKNN 后端（RK3588） python -m app.ros2.realtime_octomap_node --source 0 --backend rknn # 使用 ONNX 后端 python -m app.ros2.realtime_octomap_node --source 0 --backend onnx ``` #### 从 RTSP 流发布 ```bash python -m app.ros2.realtime_octomap_node \ --source rtsp://192.168.1.100:554/stream \ --backend rknn ``` #### 从视频文件发布 ```bash python -m app.ros2.realtime_octomap_node \ --source video.mp4 \ --backend onnx ``` #### 从 ROS2 图像话题订阅 ```bash python -m app.ros2.realtime_octomap_node \ --source-topic /camera/image_raw \ --backend onnx ``` #### 启用 MJPEG 流输出 ```bash python -m app.ros2.realtime_octomap_node \ --source 0 \ --backend rknn \ --mjpeg \ --mjpeg-host 0.0.0.0 \ --mjpeg-port 8080 # 在浏览器中访问：http://<设备 IP>:8080/stream.mjpg ``` #### 保存结果 ```bash python -m app.ros2.realtime_octomap_node \ --source 0 \ --backend rknn \ --save \ --project runs/realtime ``` ### 参数 | 参数 | 说明 | 默认值 | |------|------|--------| | `--source` | 输入源（摄像头索引/视频文件/RTSP 流） | `None` | | `--source-topic` | ROS2 图像话题 | `None` | | `--weights` | 权重文件路径（.onnx 或.rknn） | `None` | | `--data` | 数据配置 YAML | `None` | | `--backend` | 推理后端（auto/onnx/rknn） | `auto` | | `--device` | 推理设备 | 默认设备 | | `--conf-thres` | 置信度阈值 | `0.25` | | `--iou-thres` | NMS IoU 阈值 | `0.45` | | `--imgsz` | 推理尺寸 | `640` | | `--grid-scale` | 栅格缩放 | `8` | | `--frame-id` | TF 坐标系 ID | `map` | | `--resolution` | 地图分辨率 | `1.0` | | `--mjpeg` | 启用 MJPEG 流 | `False` | | `--mjpeg-host` | MJPEG 服务器地址 | `0.0.0.0` | | `--mjpeg-port` | MJPEG 服务器端口 | `8080` | | `--save` | 保存结果 | `False` | | `--project` | 项目目录 | `None` | --- ## 发布的话题 | 话题名称 | 消息类型 | 说明 | |---------|---------|------| | `/yoseg/octomap` | `octomap_msgs/Octomap` | 3D 八叉树地图（主推） | | `/yoseg/occ2d_grid` | `nav_msgs/OccupancyGrid` | 2D 占据栅格（可选） | | `/yoseg/z_band_markers` | `visualization_msgs/MarkerArray` | Z 波段可视化（可选） | --- ## 使用示例 ### 一键演示（RK3588） ```bash # 进入项目目录 cd /path/to/Yosegment # 设置脚本权限 chmod +x tools/rk3588_ros2_demo.sh # 运行一键演示 ./tools/rk3588_ros2_demo.sh --all ``` ### 完整工作流程 1. **生成分割数据** ```bash python -m app.inference.segmentation --source data/images --output runs/segment/exp ``` 2. **导出为 NPZ** ```bash python -m app.mapping.octomap_export \ --mask-dir runs/segment/exp/masks \ --out-npz data/my_map.npz ``` 3. **发布到 ROS2** ```bash python -m app.ros2.occ_zband_publisher \ --npz data/my_map.npz \ --frame-id map \ --resolution 1.0 \ --publish-octomap ``` 4. **在 RViz2 中查看** ```bash source /opt/ros/humble/setup.bash rviz2 # 添加 Octomap 显示，选择 /yoseg/octomap 话题 ``` --- ## 常见问题 ### Q: 提示 "octomap" 模块未找到 ```bash pip install octomap ``` ### Q: 提示 "octomap_msgs" 未找到 ```bash # Ubuntu/Debian sudo apt install ros-humble-octomap-msgs ``` ### Q: RViz2 无法显示 Octomap 1. 检查 Fixed Frame 是否设置为 `map` 2. 确认话题 `/yoseg/octomap` 有数据：`ros2 topic echo /yoseg/octomap --once` 3. 确认 Octomap 显示已添加并订阅正确话题 --- ##
