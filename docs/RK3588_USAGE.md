# Yosegment ROS2 Octomap - RK3588 使用指南 本文档介绍如何在 RK3588（Ubuntu 系统）上运行 Yosegment 的 ROS2 Octomap 功能。 --- ## 1. 快速开始 ### 1.1 环境准备 确保 RK3588 已安装以下依赖： ```bash # 1. 安装 ROS2 Humble（如果尚未安装） sudo apt update sudo apt install ros-humble-desktop ros-humble-octomap-msgs ros-humble-octomap-ros # 2. 安装 Python 依赖 pip3 install octomap numpy # 3. 克隆项目（如果尚未克隆） git clone <repository-url> cd Yosegment ``` ### 1.2 一键演示 最简单的方式是运行一键演示脚本： ```bash # 进入项目目录 cd /path/to/Yosegment # 设置脚本权限 chmod +x tools/rk3588_ros2_demo.sh # 运行一键演示 ./tools/rk3588_ros2_demo.sh --all ``` --- ## 2. 脚本使用说明 ### 2.1 帮助信息 ```bash ./tools/rk3588_ros2_demo.sh --help ``` 输出： ``` Yosegment ROS2 Octomap Demo - RK3588 操作指南 ============================================ 用法：rk3588_ros2_demo.sh [选项] 选项： -h, --help 显示帮助信息 -c, --check 仅检查环境 -p, --publish 发布示例 Octomap 数据 -s, --subscribe 订阅 Octomap 话题 -l, --list 列出所有 yoseg 相关话题 -r, --rviz 启动 RViz2 查看地图 -a, --all 一键演示（发布 + 订阅 + RViz） ``` ### 2.2 检查环境 首次运行前，建议先检查环境： ```bash ./tools/rk3588_ros2_demo.sh --check ``` 预期输出： ``` [INFO] 检查环境... [SUCCESS] 环境检查通过 [INFO] 检查 Python 依赖... [SUCCESS] 依赖检查完成 ``` ### 2.3 生成并示例数据 生成示例地图数据： ```bash python3 tools/generate_sample_map.py --out data/sample_map.npz ``` 预期输出： ``` 生成示例地图数据... 网格大小：64x64 网格比例：1.0 随机种子：42 已保存：data/sample_map.npz occ2d: shape=(64, 64), dtype=uint8, occupied=1234 z_band2d: shape=(64, 64, 2), dtype=float32 高度统计：min=0.50m, max=4.95m, mean=2.15m ``` ### 2.4 发布 Octomap 数据 发布示例数据到 ROS2： ```bash ./tools/rk3588_ros2_demo.sh --publish ``` 或使用 Python 直接运行： ```bash source /opt/ros/humble/setup.bash python3 app/ros2/occ_zband_publisher.py \ --npz data/sample_map.npz \ --frame-id map \ --resolution 1.0 \ --rate 2.0 \ --publish-octomap ``` ### 2.5 订阅话题查看数据 在新终端订阅话题： ```bash ./tools/rk3588_ros2_demo.sh --subscribe ``` 或使用 ROS2 命令： ```bash source /opt/ros/humble/setup.bash ros2 topic echo /yoseg/octomap --once ``` ### 2.6 查看可用话题 列出所有 Yosegment 相关话题： ```bash ./tools/rk3588_ros2_demo.sh --list ``` 或： ```bash ros2 topic list | grep yoseg ``` 预期输出： ``` /yoseg/octomap /yoseg/occ2d_grid /yoseg/z_band_markers ``` --- ## 3. RViz2 可视化 ### 3.1 启动 RViz2 ```bash ./tools/rk3588_ros2_demo.sh --rviz ``` 或手动启动： ```bash source /opt/ros/humble/setup.bash rviz2 ``` ### 3.2 添加 Octomap 显示 1. 打开 RViz2 2. 点击 "Add" 按钮 3. 选择 "By topic" 标签 4. 找到 `/yoseg/octomap` 话题 5. 点击 "OK" 添加 ### 3.3 RK3588 性能优化建议 由于 RK3588 的 GPU 性能限制，建议： 1. 使用较低的地图分辨率（如 0.5m 或 1.0m） 2. 减少地图尺寸（如 32x32 或 64x64） 3. 使用轻量级查看方式（如 `ros2 topic echo`） --- ## 4. 完整工作流程示例 ### 4.1 从分割到 Octomap 发布 ```bash # 步骤 1: 运行分割（如果有图像数据） python3 -m app.inference.segmentation --load-masks # 步骤 2: 导出为 NPZ python3 -m app.mapping.octomap_export \ --mask-dir runs/segment/exp/masks \ --out-npz data/my_map.npz # 步骤 3: 发布到 ROS2 python3 -m app.ros2.occ_zband_publisher \ --npz data/my_map.npz \ --frame-id map \ --resolution 1.0 \ --rate 2.0 \ --publish-octomap ``` ### 4.2 实时订阅（配合其他节点） ```bash # 启动实时订阅节点 python3 -m app.ros2.realtime_occ_zband_node \ --sub-topic /yoseg/octomap_snapshot_json \ --frame-id map \ --resolution 1.0 \ --publish-octomap ``` --- ## 5. 常见问题 ### Q1: 提示 "octomap" 模块未找到 ```bash pip3 install octomap ``` ### Q2: ROS2 话题列表为空 确保已 source ROS2 环境： ```bash source /opt/ros/humble/setup.bash ``` ### Q3: RViz2 无法显示 Octomap 检查： 1. Fixed Frame 是否设置为 "map" 2. Octomap 话题是否正确订阅 3. 消息是否正常发布（使用 `ros2 topic echo` 验证） ### Q4: RK3588 运行缓慢 - 降低地图分辨率 - 减少发布频率 - 使用无头模式（不启动 RViz2） --- ## 6. 参考链接 - [ROS2 Humble 文档](https://docs.ros.org/en/humble/) - [OctoMap 官方文档](https://octomap.github.io/) - [Yosegment 项目文档](../README.md)
